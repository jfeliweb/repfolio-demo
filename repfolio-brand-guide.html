Logo Usage Rules

✓ Do This

  • Use two-color version on light backgrounds
  • Maintain minimum clear space (25% of logo height)
  • Scale proportionally, never distort
  • Use single-color at sizes below 40px
  • Ensure minimum 24px height on mobile
  • Test at 1x, 2x, 3x screen densities

✗ Don't Do This

  • Stretch or distort the logo
  • Add shadows, outlines, or gradients
  • Change letter spacing
  • Use on busy or low-contrast backgrounds
  • Rotate or skew the logo
  • Change the color combinations

Typography

System font stack for instant loading and native feel across all platforms. Responsive sizing ensures readability from mobile to desktop.

Display Heading
font-size: clamp(2rem, 5vw, 3rem); font-weight: 700;

Use for hero sections, landing pages, major section titles. Min 32px, max 48px.

Section Heading (H2)
font-size: clamp(1.5rem, 4vw, 2rem); font-weight: 600;

Main content sections. Min 24px, max 32px.

Subsection Heading (H3)
font-size: clamp(1.25rem, 3vw, 1.5rem); font-weight: 600;

Card headers, subsections. Min 20px, max 24px.

Body text maintains excellent readability at 16px with 1.6 line height. This is the primary text size used throughout the application for paragraphs, descriptions, and general content. Never go below 16px for body text on mobile.
font-size: 1rem (16px); line-height: 1.6;
Small text at 14px for captions, metadata, and secondary information. Use sparingly and never for critical content.
font-size: 0.875rem (14px); color: secondary;

Accessibility Checklist

Touch Targets

  • Minimum 44×44px (iOS standard)
  • Minimum 48×48px (Android Material)
  • Adequate spacing between elements (8px+)
  • Visual feedback on all interactions

Keyboard Navigation

  • All interactive elements keyboard accessible
  • Visible focus indicators (2px outline)
  • Logical tab order throughout app
  • Skip links for main content

Screen Readers

  • Semantic HTML (nav, main, section)
  • ARIA labels for icons and buttons
  • Alt text for all images
  • Descriptive link text (no "click here")

Visual Design

  • Don't rely on color alone
  • Minimum 16px font size for body text
  • 1.5+ line height for readability
  • Test with color blindness simulators

Mobile-First Design

Design for mobile first, then enhance for larger screens. Mobile users are your primary audience—coaches checking schedules on the field, parents viewing updates during games.

Layout

  • Single column on mobile, multi-column on tablet+
  • Stack navigation vertically on small screens
  • Bottom navigation for primary actions (iOS pattern)
  • Generous padding (16px minimum)
  • Full-width touch targets on mobile

Performance

  • Optimize images for mobile (WebP format)
  • Lazy load below-the-fold content
  • Minimize JavaScript bundle size
  • System fonts (zero download time)
  • Progressive enhancement approach

Interaction

  • Large, easy-to-tap buttons (44px min)
  • Swipe gestures for common actions
  • Pull-to-refresh for list views
  • Haptic feedback on actions (iOS)
  • Loading states for all async operations

Content

  • Prioritize essential information
  • Progressive disclosure for complex data
  • Readable at arm's length (16px+ text)
  • Thumb-friendly zone for primary actions
  • Minimize form fields on mobile

Brand Voice & Messaging

Repfolio bridges the gap between professional business tools and personal sports development. Your voice should be confident but approachable, efficient but caring.

Professional

  • "Manage your coaching business efficiently"
  • "Track progress with detailed analytics"
  • "Streamline payments and scheduling"
  • Focus on reliability, security, business value

Friendly

  • "Watch your athletes grow and improve"
  • "Celebrate every milestone together"
  • "Keep parents in the loop automatically"
  • Focus on development, relationships, support

Efficient

  • "Book sessions in seconds, not minutes"
  • "Weather cancellations? We've got you"
  • "One tap to send progress updates"
  • Focus on time-saving, automation, simplicity

Writing Style

  • Active voice: "Create your first session"
  • Clear actions: "Book Now" not "Click Here"
  • Conversational but concise
  • Avoid jargon, explain technical terms
